Setting up the Alarmpoint Integration client.



1. The Notification integration client binary is OpenviewNotificationClient. There is an instance of this binary under "/apps/esm/eeb". It takes 16 parameters:

1. Message ID
2. Node Name
3. Node Type
4. Creation Date
5. Creation Time
6. Receive Date
7. Receive Time
8. Name of application
9. Name of the message group
10. Name of the object
11. Severity of the event (1-5, with 1 being most severe)
12. Operators
13. The message of the monitoring event
14. Instructions for dealing with this monitoring event
15. Message attributes
16. Suppression count
2. The ticketing integration client binary is OpenviewTTIClient. There is an instance of this binary under "/apps/esm/eeb". It takes 16 parameters:

1. Message ID
2. Node Name
3. Node Type
4. Creation Date
5. Creation Time
6. Receive Date
7. Receive Time
8. Name of application
9. Name of the message group
10. Name of the object
11. Severity of the event (1-5, with 1 being most severe)
12. Operators
13. The message of the monitoring event
14. Instructions for dealing with this monitoring event
15. Message attributes
16. Suppression count
